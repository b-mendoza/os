# Workflow

## Keep Solutions Small

- Prefer the simplest implementation that satisfies the current task.
- Avoid abstractions for hypothetical future requirements.
- If a solution is hard to explain quickly, simplify it before expanding it.

## Task Management

- Create a GitHub issue for multi-step or non-trivial work that needs tracking across sessions or handoffs.

## Task Decomposition

- Break large tasks into smaller, independently reviewable and testable units.
- Each subtask should have a clear definition of done.
- Avoid unnecessary blocking dependencies between subtasks.
