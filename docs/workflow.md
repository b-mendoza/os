# Workflow Guidelines

## Simplicity First

Never implement complex solutions to simple problems.

Before implementing any solution, ask:

- Can this be achieved with fewer abstractions?
- Am I over-engineering for hypothetical future requirements?
- Would a junior developer understand this in 5 minutes?

## Task Management

Create a GitHub issue for any multi-step or non-trivial task.

**Benefits:**

- Tracks progress across sessions
- Enables structured, organized completion
- Creates documentation for future reference
- Allows work to be paused and resumed without context loss

## Task Decomposition

Break large tasks into smaller, independently mergeable and testable units.

**Well-scoped subtask criteria:**

- Can be reviewed in isolation
- Has a clear definition of "done"
- Doesn't create blocking dependencies
- Can be safely merged independently
